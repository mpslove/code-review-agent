🔴 **[CRITICAL]** `httpx/dispatch/http2.py:127` — Missing timeout in flow control wait

> In send_data(), when the flow control window is zero, the code waits on window_update_received[stream_id].wait() without any timeout. This can cause a permanent hang if the server never sends a WINDOW_UPDATE frame (e.g., due to a bug or malicious behavior). The existing timeout parameter passed to send_data is ignored.

<details>
<summary>💡 Suggestion</summary>

