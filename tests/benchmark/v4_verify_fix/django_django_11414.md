### Performance Review (1 consensus)

🟡 **[MEDIUM]** `django/http/response.py:475` — 大文件场景下低效的Content-Length计算 `[2/2 rounds]`
> 对于没有`tell()`但可`seek()`的文件对象，`set_headers`使用`sum(iter(lambda: len(filelike.read(self.block_size)), 0))`逐块读取整个文件内容来计算长度。该方式会分配并丢弃大量小块bytes对象（每个`self.block_size`约64KB），导致高内存分配和GC开销。对于大文件（如数百MB），此操作将完全扫描文
<details><summary>💡 Suggestion</summary>

```
将低效读取方式改为仅用seek操作计算长度：
```python
elif seekable:
    # 对于没有tell但可seek的对象，通过seek获取大小
    current = filelike.seek(0, io.SEEK_CUR) if hasattr(filelike, 'seek') and callable(filelike.seek) else 0
    filelike.seek(0, io.SEEK_END)
    end = filelike.tell() if hasattr(filelike, 'tell') else filelike.seek(0, io.SEEK_CUR)
    self.headers['Content-Length'] = end - current
    filelike.seek(current)
```
若对象没有`tell()`且`seek`也不返回新位置（极少见），可保留原方案但增加大文件警告。
```
